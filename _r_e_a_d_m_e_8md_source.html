25 ### <span id="0_0_1">What is EPICS base?</span>
26 
27 The Experimental Physics and Industrial Control Systems (EPICS) is an
28 extensible set of software components and tools with which application
29 developers can create a control system. This control system can be
30 used to control accelerators, detectors, telescopes, or other
31 scientific experimental equipment. EPICS base is the set of core
32 software, i.e. the components of EPICS without which EPICS would not
33 function. EPICS base allows an arbitrary number of target systems,
34 IOCs (input/output controllers), and host systems, OPIs (operator
35 interfaces) of various types.

47 ### <span id="0_0_4">Supported platforms</span>
48 
49 The list of platforms supported by this version of EPICS base is given
50 in the `configure/CONFIG_SITE` file. If you are trying to build EPICS
51 Base on an unlisted host or for a different target machine you must
52 have the proper host/target cross compiler and header files, and you
53 will have to create and add the appropriate new configure files to the
54 base/configure/os/directory. You can start by copying existing
55 configuration files in the configure/os directory and then make
56 changes for your new platforms.

58 ### <span id="0_0_5">Supported compilers</span>
59 
60 This version of EPICS base has been built and tested using the host
61 vendor's C and C++ compilers, as well as the GNU gcc and g++
62 compilers. The GNU cross-compilers work for all cross-compiled
63 targets. You may need the C and C++ compilers to be in your search
64 path to do EPICS builds; check the definitions of CC and CCC in
65 `base/configure/os/CONFIG.<host>.<host>` if you have problems.

67 ### <span id="0_0_6">Software requirements</span>
68 
69 **GNU make**
70 You must use GNU make, gnumake, for any EPICS builds. Set your path so
71 that a gnumake version 3.81 or later is available.
72 
73 **Perl**
74 You must have Perl version 5.8.1 or later installed. The EPICS
75 configuration files do not specify the perl full pathname, so the perl
76 executable must be found through your normal search path.
77 
78 **Unzip and tar (Winzip on WIN32 systems)**
79 You must have tools available to unzip and untar the EPICS base
80 distribution file.
81 
82 **Target systems**
83 EPICS supports IOCs running on embedded platforms such as VxWorks and
84 RTEMS built using a cross-compiler, and also supports soft IOCs
85 running as processes on the host platform.
86 
87 **vxWorks**
88 You must have vxWorks 6.8 or later installed if any of your target
89 systems are vxWorks systems; the C++ compiler from older versions cannot
90 compile recently developed code. The vxWorks installation provides the
91 cross-compiler and header files needed to build for these targets. The
92 absolute path to and the version number of the vxWorks installation
93 must be set in the `base/configure/os/CONFIG_SITE.Common.vxWorksCommon`
94 file or in one of its target-specific overrides.
95 
96 Consult the [vxWorks 6.x](https://epics.anl.gov/base/vxWorks6.php) EPICS
97 web pages about and the vxWorks documentation for information about
98 configuring your vxWorks operating system for use with EPICS.
99 
100 **RTEMS**
101 For RTEMS targets, you need RTEMS core and toolset version 4.9.x or
102 4.10.x (4.11 or 5.x are not yet supported).
103 
104 **GNU readline or Tecla library**
105 GNU readline and Tecla libraries can be used by the IOC shell to
106 provide command line editing and command line history recall and edit.
107 GNU readline (or Tecla library) must be installed on your target
108 system when `COMMANDLINE_LIBRARY` is set to READLINE (or TECLA) for
109 that target. EPICS (EPICS shell) is the default specified in
110 `CONFIG_COMMON`. A READLINE override is defined for linux-x86 in the
111 EPICS distribution. Comment out `COMMANDLINE_LIBRARY=READLINE` in
112 `configure/os/CONFIG_SITE.Common.linux-x86` if readline is not
113 installed on linux-x86. Command-line editing and history will then be
114 those supplied by the os. On vxWorks the ledLib command-line input
115 library is used instead.

234 #### Set environment variables
235 
236 Files in the base/startup directory have been provided to help set
237 required path and other environment variables.
238 
239 * `EPICS_HOST_ARCH`
240 Before you can build or use EPICS R3.15, the environment variable
241 `EPICS_HOST_ARCH` must be defined. A perl script EpicsHostArch.pl in
242 the base/startup directory has been provided to help set
243 `EPICS_HOST_ARCH.` You should have `EPICS_HOST_ARCH` set to your
244 host operating system followed by a dash and then your host
245 architecture, e.g. solaris-sparc. If you are not using the OS
246 vendor's c/c++ compiler for host builds, you will need another dash
247 followed by the alternate compiler name (e.g. "-gnu" for GNU c/c++
248 compilers on a solaris host or "-mingw" for MinGW c/c++ compilers on
249 a WIN32 host). See `configure/CONFIG_SITE` for a list of supported
250 `EPICS_HOST_ARCH` values.
251 
252 * `PERLLIB`
253 On WIN32, some versions of Perl require that the environment
254 variable PERLLIB be set to &lt;perl directory location>.
255 
256 * `PATH`
257 As already mentioned, you must have the perl executable and you may
258 need C and C++ compilers in your search path. For building base you
259 also must have echo in your search path. For Unix host builds you
260 also need ln, cpp, cp, rm, mv, and mkdir in your search path and
261 /bin/chmod must exist. On some Unix systems you may also need ar and
262 ranlib in your path, and the C compiler may require as and ld in
263 your path. On solaris systems you need uname in your path.
264 
265 * `LD_LIBRARY_PATH`
266 R3.15 shared libraries and executables normally contain the full
267 path to any libraries they require. However, if you move the EPICS
268 files or directories from their build-time location then in order
269 for the shared libraries to be found at runtime `LD_LIBRARY_PATH`
270 must include the full pathname to
271 `$(INSTALL_LOCATION)/lib/$(EPICS_HOST_ARCH)` when invoking
272 executables, or some equivalent OS-specific mechanism (such as
273 /etc/ld.so.conf on Linux) must be used. Shared libraries are now
274 built by default on all Unix type hosts.

276 #### Do site-specific build configuration
277 
278 **Site configuration**
279 To configure EPICS, you may want to modify the default definitions
280 in the following files:
281 
282 ```
283  configure/CONFIG_SITE Build choices. Specify target archs.
284  configure/CONFIG_SITE_ENV Environment variable defaults
285  configure/RELEASE TORNADO2 full path location
286 ```
287 
288 **Host configuration**
289 To configure each host system, you may override the default
290 definitions by adding a new file in the configure/os directory with
291 override definitions. The new file should have the same name as the
292 distribution file to be overridden except with CONFIG in the name
293 changed to `CONFIG_SITE`.
294 
295 ```
296  configure/os/CONFIG.<host>.<host> Host build settings
297  configure/os/CONFIG.<host>.Common Host common build settings
298 ```
299 
300 **Target configuration**
301 To configure each target system, you may override the default
302 definitions by adding a new file in the configure/os directory with
303 override definitions. The new file should have the same name as the
304 distribution file to be overridden except with CONFIG in the name
305 replaced by `CONFIG_SITE`. This step is necessary even if the host
306 system is the only target system.
307 
308 ```
309  configure/os/CONFIG.Common.<target> Target common settings
310  configure/os/CONFIG.<host>.<target> Host-target settings
311 ```

313 #### Build EPICS base
314 
315 After configuring the build you should be able to build EPICS base
316 by issuing the following commands in the distribution's root
317 directory (base):
318 
319 ```
320  gnumake clean uninstall
321  gnumake
322 ```
323 
324 The command "gnumake clean uninstall" will remove all files and
325 directories generated by a previous build. The command "gnumake"
326 will build and install everything for the configured host and
327 targets.
328 
329 It is recommended that you do a "gnumake clean uninstall" at the
330 root directory of an EPICS directory structure before each complete
331 rebuild to ensure that all components will be rebuilt.

333 ### <span id="0_0_13">Example application and extension</span>
334 
335 A perl tool, makeBaseApp.pl is included in the distribution file. This
336 script will create a sample application that can be built and then
337 executed to try out this release of base.
338 
339 Instructions for building and executing the 3.15 example application
340 can be found in the section "Example Application" of Chapter 2,
341 "Getting Started", in the "IOC Application Developer's Guide" for this
342 release. The "Example IOC Application" section briefly explains how to
343 create and build an example application in a user created &lt;top>
344 directory. It also explains how to run the example application on a
345 vxWorks ioc or as a process on the host system. By running the example
346 application as a host-based IOC, you will be able to quickly implement
347 a complete EPICS system and be able to run channel access clients on
348 the host system.
349 
350 A perl script, makeBaseExt.pl, is included in the distribution file.
351 This script will create a sample extension that can be built and
352 executed. The makeBaseApp.pl and makeBaseExt.pl scripts are installed
353 into the install location `bin/<hostarch>` directory during the base
354 build.

356 ### <span id="0_0_14">Multiple host platforms</span>
357 
358 You can build using a single EPICS directory structure on multiple
359 host systems and for multiple cross target systems. The intermediate
360 and binary files generated by the build will be created in separate
361 subdirectories and installed into the appropriate separate host/target
362 install directories. EPICS executables and perl scripts are installed
363 into the `$(INSTALL_LOCATION)/bin/<arch>` directories. Libraries are
364 installed into $`(INSTALL_LOCATION)/lib/<arch>`. The default
365 definition for `$(INSTALL_LOCATION)` is `$(TOP)` which is the root
366 directory in the distribution directory structure, base. Created
367 object files are stored in `O.<arch>` source subdirectories, This
368 allows objects for multiple cross target architectures to be
369 maintained at the same time. To build EPICS base for a specific
370 host/target combination you must have the proper host/target C/C++
371 cross compiler and target header files and the base/configure/os
372 directory must have the appropriate configure files.
